# 深圳合租点推荐器

一个面向深圳合租场景的前端 Demo：当多位合租成员分别在不同地点上班或上学时，系统根据通勤公平性、平均通勤时间、最差通勤惩罚、租金适配、生活便利和地铁可达性，为他们推荐更适合一起居住的合租区域。

在线访问地址：

https://xun98686.github.io/shenzhen-rental-demo/

## 项目背景

深圳合租决策通常不是单人决策，而是多人协商。每个人的工作或学习地点不同，如果只看“平均通勤”，可能会让其中一个人承担明显更差的通勤成本。

这个 Demo 的核心目标是把“哪里适合一起住”从主观讨论变成可解释的推荐结果：

- 谁的通勤最远？
- 哪个区域对所有人更公平？
- 哪些区域租金更友好？
- 如果偏好从“通勤优先”改成“租金优先”，推荐结果是否变化？

## 核心功能

- 合租成员配置：支持 2-4 位成员，成员可以选择自己的上班或上学地点。
- 通勤上限设置：每个成员可以设置自己可接受的最大通勤时长。
- 推荐偏好调节：支持调整通勤公平、平均通勤、最差通勤、租金适配、生活便利、地铁可达的权重。
- 区域推荐排序：系统会对深圳多个候选合租区域打分，并输出推荐排名。
- 通勤示意图：页面展示候选区域、成员目的地和模拟通勤路线。
- 结果解释：每个推荐区域会给出平均通勤、最长通勤、通勤差距、租金区间和推荐理由。

## 推荐算法

当前推荐分数由六类因素组成：

```text
总分 = 通勤公平 + 平均通勤 + 最差通勤惩罚 + 租金适配 + 生活便利 + 地铁可达
```

主要规则：

- 通勤公平：成员之间通勤时间差距越小，分数越高。
- 平均通勤：所有成员平均通勤越短，分数越高。
- 最差通勤惩罚：如果某个成员通勤特别长，或超过其可接受上限，会降低推荐分。
- 租金适配：根据候选区域的租金友好程度计分。
- 生活便利：根据候选区域生活配套评分。
- 地铁可达：根据候选区域轨道交通便利程度评分。

这个规则更适合多人合租，因为它不是单纯追求最短平均通勤，而是兼顾“大家是否都能接受”。

## 地图 API 设计

Demo 中使用 `DemoMapAdapter` 模拟地图服务的通勤矩阵能力，包括：

- 地点搜索
- 区域到目的地的通勤时长
- 通勤距离
- 路线点位
- 多起点、多终点的通勤矩阵

真实上线时，可以把 `DemoMapAdapter` 替换为高德地图、腾讯地图或百度地图的真实 Adapter。推荐算法层不需要改动，只需要保证真实地图 Adapter 返回相同结构的数据。

## 部署方式

本项目通过 GitHub Actions 自动部署到 GitHub Pages。

部署流程：

1. 推送代码到 `main` 分支。
2. GitHub Actions 自动运行测试。
3. 测试通过后，将 `app/` 目录作为静态站点发布到 GitHub Pages。

发布配置文件：

```text
.github/workflows/pages.yml
```

## 设计说明

这个 Demo 关注多人合租决策中的公平性问题。它不是只推荐离某一个人最近的地方，而是综合多个人的通勤时间、可接受上限、租金和生活便利度，给出一个可解释、可调整权重的推荐结果。

当前版本使用 `DemoMapAdapter` 模拟地图通勤矩阵。真实上线时可以替换为 `AMapAdapter` 或其他地图服务 Adapter，推荐算法和前端交互可以保持不变。
